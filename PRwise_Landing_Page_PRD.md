### 3.4 产品价值 Section (The Solution)

**标题**: "AI 做初筛，人来决策"

**How It Works (3步)**:

```
① 提交 PR       ② AI 自动审查       ③ 结果发布
     │                │                  │
     ▼                ▼                  ▼
┌─────────┐   ┌──────────────┐   ┌──────────────┐
│ 开发者    │   │ PRwise 读取   │   │ PR 下方直接   │
│ 提交 PR   │ → │ diff + 上下文 │ → │ 显示审查结果   │
│ 到 GitHub │   │ + 相关文件    │   │ + 修改建议     │
└─────────┘   └──────────────┘   └──────────────┘
```

**关键差异化**:
- "不是简单的 Lint 工具，而是理解你代码逻辑的 AI"
- "支持增量 Review：只审查变更部分，不重复检查"
- "可调节敏感度：从『只找明确 Bug』到『严格风格审查』"

### 3.7 实时示例 Section (Live Demo)

**标题**: "看它如何工作"

**展示方式** (选 1 种):

**方案 A — 静态示例（MVP 首选）**:
- 左侧展示一段有 Bug 的 PR Diff（高亮红色）
- 右侧展示 AI 生成的 Review Comment
- 下方展示修复后的代码（绿色）

示例展示内容:
```
PR 变更: user.ts :45-48
──────────────────────────────────────────
  function getUser(id) {
-   const data = fetch(`/users/${id}`);
+   if (!id) throw new Error('id is required');
+   const data = await fetch(`/users/${id}`);
    return data.json();
  }

PRwise 发现:
⚠️ 缺少 await — fetch() 返回 Promise，需要 await 才能拿到结果
⚠️ 缺少空值检查 — id 可能为空或 undefined
💡 建议: 添加参数校验 + await 关键字
```

- 文本框中粘贴代码或 PR 链接

### 3.10 FAQ Section

**6-8 个常见问题**:

| 问题 | 回答 |
|------|------|
| 我的代码会被用来训练模型吗？ | **不会**。我们仅使用 LLM API 进行推理，不会将你的代码用于训练。代码通过 TLS 加密传输，不留存在我们服务器上。 |
| 支持自托管吗？ | 当前为 SaaS。企业版支持私有部署（2026 Q4 计划中）。 |
| 准确率怎么样？ | 内部测试显示 Bug 检出率 > 85%，误报率 < 15%。你可以调节敏感度来平衡准确率和召回率。 |
| 会替换掉人工 Code Review 吗？ | **不会**。我们的定位是 AI 初筛 + 人工决策。AI 处理常规问题，人类关注架构、设计和业务逻辑。 |
| 支持 GitLab / Bitbucket 吗？ | GitHub 优先。GitLab 正在开发中（预计 2026 Q3）。 |
| 免费版限制 50 次/月，是一次 Review 算一次吗？ | 是的。每次 PR 提交触发一次 Review 消耗一次额度。 |
| 我的团队超过 10 人怎么办？ | Team 版支持 10 人。更大团队请[联系我们]获取定制方案。 |
| 可以只对特定目录/文件做 Review 吗？ | 可以。在自定义规则中配置 include/exclude 路径模式。 |
